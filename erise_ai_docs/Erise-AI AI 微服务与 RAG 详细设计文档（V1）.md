# Erise-AI AI 微服务与 RAG 详细设计文档（V1）

## 1. 定位
AI 微服务是围绕知识库、严格受权限约束、可调用受控工具的专属 AI 服务层，不是通用聊天后端。

## 2. 核心能力
- 项目内知识问答
- 文件/文档总结
- 带引用回答
- AI 会话管理
- Tool Calling
- MCP 扩展

## 3. RAG 流程
```text
用户问题
 -> 权限过滤
 -> 查询改写
 -> 检索召回
 -> 重排
 -> 上下文组装
 -> Prompt 构建
 -> 模型生成
 -> 输出答案 + citations + usedTools
```

## 4. 知识入库流程
```text
文件/文档创建或更新
 -> 文本抽取
 -> 清洗
 -> 分块
 -> 生成 embedding
 -> 写入向量库
 -> 写入引用元数据
```

## 5. 分块与检索策略
- 优先按标题层级 + 段落切分
- PDF 保留页码
- 文档保留 section_path / chunk_index
- V1 推荐关键词 + 向量混合召回 + TopK 重排

## 6. 输出结构
- answer
- citations
- usedTools
- confidence
- refusedReason

## 7. 工具调用设计
### 工具分类
- 查询工具：项目/文件/文档/搜索
- 轻写入工具：文档草稿、标签写入
- 管理工具：任务状态、系统统计

### 原则
- 工具白名单
- JSON Schema 参数校验
- 权限校验
- 审计日志
- 结果脱敏

## 8. SQL 工具限制
- 仅 SELECT
- 白名单表或视图
- 限制结果行数
- 禁止多语句
- 必须审计

## 9. MCP 使用建议
- 核心业务工具优先内部 Tool API
- 外部扩展工具通过 MCP 接入
- MCP 工具同样纳入权限与审计体系

## 10. 安全要求
- AI 不得越权检索
- Prompt 注入防护
- 无依据时拒答或降级回答
- 不得伪造引用
