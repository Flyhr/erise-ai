# Erise-AI 权限模型与安全设计文档（V1）

## 1. 权限模型
V1 采用：
- RBAC（角色权限）
- Ownership（资源归属权限）
- Authentication（接口鉴权）
- Audit（审计追踪）

## 2. 角色定义
- USER：普通用户
- ADMIN：管理员

## 3. 核心资源
- 项目
- 文件
- 文档
- 标签
- AI 会话
- 知识块
- 任务
- 审计日志
- 系统配置

## 4. 权限规则
- 未登录不得访问业务接口
- 非管理员不得访问 `/admin/**`
- 非资源拥有者不得访问对应资源
- 搜索与 AI 检索都必须先做权限过滤

## 5. 鉴权设计
- Spring Security + JWT + Refresh Token
- Access Token 包含 userId / username / roleCode
- WebSocket 连接必须校验 token

## 6. 分层实现
- Controller 不做复杂权限逻辑
- Service 负责角色校验、资源归属校验、事务控制
- Mapper 不承担权限逻辑

## 7. 文件安全
- 上传校验大小 / 类型 / MIME
- 下载前校验归属
- 预览仅在授权后返回
- 不暴露永久公网对象地址

## 8. 搜索安全
- 搜索前限制用户可访问的项目与资源范围
- 不返回无权限资源的标题、片段、项目信息

## 9. AI 安全
- AI 不得绕过业务权限
- Tool Calling 必须白名单 + 参数校验 + 权限校验 + 审计
- SQL 工具只读、限表、限结果
- 无依据时禁止编造答案与引用

## 10. 审计要求
必须审计：
- 登录/登出
- 密码修改
- 文件上传/下载/删除
- 文档发布/删除
- 管理员配置修改
- AI 工具调用
- SQL 工具调用
