# 15. 上线清单

- 版本：前端闭环与验收收口批次
- 日期：2026-04-18
- 适用范围：`erise-ai-ui`、AI 接口、索引任务、MCP / n8n 外围流程

## 15.1 发布前检查

### 环境与配置

- 确认 `VITE_API_BASE_URL` 指向目标网关地址。
- 确认 Nginx 使用 Docker DNS 动态解析后端 upstream，后端容器重启后 `/api/v1/auth/captcha` 与 `/actuator/health` 仍为 200。
- 确认 AI Provider 对应模型配置已在后台启用，且默认模型可用。
- 确认文件解析 / 索引任务依赖服务可用。
- 确认 MCP 代理、n8n Webhook、审批 / 通知 / 巡检相关环境变量已注入目标环境。

### 账号与权限

- 准备一套普通用户账号，一套管理员账号。
- 普通用户只能访问工作台、项目、文档、文件、搜索、AI，不应进入 `/admin`。
- 管理员应能从工作台导航进入管理后台，并访问 `/admin/acceptance`。

## 15.2 自动化回归

- 在 `erise-ai-ui` 下执行 `npm run build`。
- 在 `erise-ai-ui` 下执行 `npm run e2e -- tests/e2e/acceptance.spec.ts`。
- 若本机未安装 Playwright 浏览器，优先复用本机 Chrome；当前配置默认使用 `PLAYWRIGHT_BROWSER_CHANNEL=chrome`。
- 若目标网关不是 `http://127.0.0.1:8088`，通过 `PLAYWRIGHT_BASE_URL` 与 `PLAYWRIGHT_API_BASE_URL` 指向目标环境。

## 15.3 发布前冒烟顺序

1. 普通用户登录，确认落地到 `/workspace`。
2. 打开 `/projects`，进入任意项目详情，再进入项目 AI。
3. 在 `/documents` 创建或打开文档，验证预览、编辑、删除确认。
4. 在 `/files` 上传一个文件，观察解析 / 索引状态推进。
5. 在 `/search` 用项目过滤执行一次搜索，验证结果跳转。
6. 在 `/ai` 发起一次问答，验证模型 / Provider 状态、引用展示、临时文件索引状态。
7. 管理员登录，确认工作台导航出现后台入口。
8. 进入 `/admin`，检查模型配置、索引任务、审计日志、前端验收页。

## 15.4 AI 与索引专项检查

- 验证 AI 页面“新建对话”确认弹层可阻止误清空。
- 验证“停止生成”确认弹层行为正确。
- 验证删除会话时会显示会话标题。
- 上传临时文件后，检查索引状态区域是否能展示“解析中 / 可引用 / 解析失败”。
- 若临时文件失败，验证“重新解析”按钮可用。
- 至少点开一条知识库引用和一条网页引用，确认跳转行为符合预期。
- 单独执行一次真实 Provider 生成冒烟，记录模型、Provider、耗时、引用数量和失败原因。

## 15.5 外围流程检查

- 抽样验证审批流至少一条。
- 抽样验证通知流至少一条。
- 抽样验证巡检 / 告警至少一条。
- 确认失败链路有记录，可在日志或告警页面追踪。
- 确认 n8n 仅承担外围工作流，不接管核心同步问答。

## 15.6 发布后观察

- 观察前端错误日志、AI 请求日志、索引任务失败率。
- 观察管理员反馈中是否出现“无法进入后台”“引用打不开”“临时文件卡住”等集中问题。
- 关注构建大包依赖带来的首屏加载波动，必要时安排后续拆包优化。

## 15.7 回滚准备

- 保留上一版前端构建产物与部署记录。
- 保留上一版 AI 模型默认配置快照。
- 若出现 AI 主路径不可用，优先回退前端静态资源，再检查 Provider 配置与索引任务服务。
