# 14. 前端闭环与验收报告

- 更新日期：2026-04-18
- 范围：`erise-ai-ui`
- 目标：补齐 AI 会话闭环、管理员入口、项目到 AI 路径，以及统一的前端验收入口，确保工作台、项目、文档、文件、搜索、AI、管理后台关键路径可验收。

## 14.1 本轮补齐项

### AI 会话闭环

- AI 页面新增会话元信息卡片，展示当前会话编号、绑定项目、消息数、最近活动时间。
- AI 页面新增模型 / Provider 状态卡片，展示当前模型、Provider、流式模式、上下文窗口和最近一次回复元信息。
- AI 页面新增索引状态卡片，集中展示已附加知识库上下文、临时文件数量、可引用上下文数、联网搜索状态。
- 临时文件列表支持直接查看解析 / 索引状态、失败重试、移除操作。
- 助理消息头补充 Provider、模型编号、引用数量等元信息。
- 引用卡片补充摘要片段与匹配信息，网页引用和知识库引用均可直接识别来源。

### 确认与安全反馈

- 新建对话前新增确认弹层，避免误清空当前消息、临时文件和草稿。
- 停止生成前新增确认弹层，明确“保留已生成内容，仅中断本轮输出”。
- 删除会话弹层改为带会话标题和影响范围说明，避免误删。

### 导航与入口

- 工作台导航为管理员新增“管理后台”入口。
- 新增项目级 AI 路由 `/projects/:id/ai`。
- 项目详情页新增“项目 AI”按钮，形成“项目 -> AI”直接闭环。
- 管理后台新增“前端闭环与验收”页面 `/admin/acceptance`，作为统一验收入口。

## 14.2 关键路径验收范围

| 模块 | 验收入口 | 核心检查点 |
| --- | --- | --- |
| 工作台 | `/workspace` | 最近项目、最近文档/文件、AI 快捷入口、空态/加载态 |
| 项目 | `/projects` -> 项目详情 -> 项目 AI | 项目概览、项目资料表、项目级 AI、资料跳转 |
| 文档 | `/documents` | 创建、筛选、编辑、预览、删除确认 |
| 文件 | `/files` | 上传、预览、在线编辑、知识解析状态、重试 |
| 搜索 | `/search` | 项目过滤、结果分组、跳转、文件状态展示 |
| AI | `/ai` | 会话元信息、模型状态、引用展示、索引状态、确认弹层 |
| 管理后台 | `/admin`、`/admin/acceptance` | 管理员入口、模型配置、索引任务、统一验收页 |

## 14.3 回归结果

### 已完成

- `npm run build` 于 2026-04-18 执行通过。
- `npm run e2e -- tests/e2e/acceptance.spec.ts` 于 2026-04-18 执行通过，覆盖普通用户关键路径与管理员后台关键路径。
- 新增页面 `AdminAcceptanceView` 已被打入构建产物。
- `WorkspaceNavigationShell`、`AiView`、`ProjectDetailView`、`router/index.ts` 修改均通过 TypeScript 编译。
- 新增 `tests/e2e/acceptance.spec.ts`，自动创建普通用户验收数据，覆盖工作台、项目、项目详情、文档、文件、搜索、AI 页面元信息 / 引用 / 索引状态 / 确认弹层。
- 管理员自动化路径覆盖 `/ai` 管理入口、`/admin`、`/admin/ai/models`、`/admin/ai/index-tasks`、`/admin/acceptance`。
- 发布演练中发现并修复 Nginx 容器重启场景下的旧 upstream IP 问题：`deploy/nginx/default.dev.conf` 与 `erise-ai-ui/nginx.conf` 已改为使用 Docker DNS 动态解析 `backend`。

### 需要带环境继续验证

- 实际 AI Provider 可用情况下的长耗时模型生成、停止生成与流式取消链路。
- 真实 RAG 同步完成后的引用跳转、网页引用与知识库引用混合展示。
- 文件上传后真实解析 / 索引状态推进与失败重试的长链路。
- MCP / n8n / 审批 / 告警的完整联调链路。

## 14.4 残余风险

- 前端构建产物中 `tinymce-core`、`echarts`、`element-plus` 体积较大，发布前需关注首屏加载和缓存策略。
- AI 页面新增状态面板后信息密度提升，需要在真实数据量较大时继续观察移动端折叠效果。
- 自动化验收中的 AI 会话详情采用前端 fixture，目的是稳定验证会话元信息、引用卡片、模型 / Provider、索引状态和确认弹层；真实 Provider 生成能力仍需按上线清单单独冒烟。

## 14.5 结论

截至 2026-04-18，本轮前端闭环项已实现并完成构建级与自动化验收验证。项目已具备统一的验收入口，工作台、项目、文档、文件、搜索、AI、管理后台关键路径均可按 Playwright 脚本复跑；上线前仍建议补做一次真实 Provider 生成、真实 RAG 引用和外围工作流联调冒烟。
